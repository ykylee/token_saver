- 결정 검증:
  - **AES-GCM**: 12-byte nonce per call (NIST SP 800-38D 권장) + AES-256 + 128-bit tag. master_key 는 32 bytes 만 허용 (AES-128 으로 silent degrade 방지)
  - **Encrypt-on-insert / decrypt-on-read**: store 가 암호화 책임지고, caller (route, registry) 는 plaintext 만 봄. Master key rotation = app restart (P1 follow-up: re-key helper)
  - **Multi-tenant isolation at store boundary**: ``user_id=None`` (admin's cross-tenant view) vs ``user_id=<id>`` (tenant filter). Route 는 role 만 보고 store 가 owner 결정
  - **Best-effort cache**: Redis 장애 시 chat_completions hot path 5xx 안 됨. Source of truth = Mongo `models_cache` sub-document
  - **Mongo index owner_user_id + composite + type**: list_for_user / list_enabled_for_user 가 sort + filter 둘 다 가능. ensure_indexes idempotent
  - **InMemoryProviderStore no encryption**: in-process working copy, 테스트 시 cipher round-trip 무관. Production 은 항상 Mongo backend
  - **chat_completions per-user registry 매 request**: registry 가 BaseProvider instance + httpx client 소유 → finally close 로 leak 방지. Optimization (cache per-user registry) 은 P1 follow-up
  - **Provider CRUD `ProviderInfo` projection**: ProviderRecord (api_key 포함) → ProviderInfo (api_key 미포함). Wire leak 방지 + invariant "API key never returned over HTTP"
  - **refresh_models delta empty on just-rewrite**: prior catalog 가 set_models_cache 호출로 overwrite 되므로 delta 가 0. Future cycle 에서 prior snapshot 별도 보관 검토 (Redis 또는 sub-document nested)
  - **redis_url default non-empty**: default "redis://localhost:6379/0" 가 tests 에서 connection fail 일으킴 → conftest 에서 redis_url="" 명시. CI / docker-compose 는 env var override

  - pyproject [tool.setuptools] packages 와 실제 디렉토리 정합 (test_package_layout_matches_pyproject 통과)
  - public module import list 와 실제 export 정합
  - dev deps: fakeredis>=2.20 추가 (in-process Redis for session + cache tests)

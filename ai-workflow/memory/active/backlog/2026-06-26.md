  - **Timing-safe dummy verify**: unknown email 분기에서 fixed hash 로 verify 1회 실행 → 양 분기 timing 비슷
  - **Lazy eviction**: SessionStore.get() 시 expires_at ≤ now 면 pop 후 None 반환 → caller 가 expiry check 안 해도 됨
  - **Protocol-first design**: SessionStore/UserStore Protocol + InMemory impl — TASK-002-4 (MongoUserStore) / TASK-002-5 (RedisSessionStore) 가 같은 Protocol 구현으로 swap 가능
  - **RBAC dependency layering**: get_current_user (parse + verify) → require_admin / require_user (role gate). 라우트별로 필요한 strictness 선택 가능
  - **/admin/* router-level dep**: admin router 자체에 `dependencies=[Depends(require_admin)]` → /admin/health 외 /admin/audit 추가 시 자동 보호
- drift 검증:
  - pyproject [tool.setuptools] packages 와 실제 디렉토리 정합 (test_package_layout_matches_pyproject 통과, 21 modules parametrised 통과)
  - create_app(settings) signature 변경 — conftest/settings fixture 가 명시적으로 settings 주입, 격리 보장
- 미실행 사유:
  - **Real Redis session_store** ❌ (TASK-002-5)
  - **Real Mongo users collection** ❌ (TASK-002-4)
  - **Bearer token rotation API** ❌ (P1)
  - **Password reset flow** ❌ (P1)
  - **Rate limit** ❌ (architecture §4.1, 별도 sub-task 미정 — TASK-002-5 또는 후속)

### 4.4 결과 (Result)

- 작업 결과:
  - Bearer token auth wire-shape lock-in — TASK-002-5 가 real session store (Redis) 추가만 하면 됨
  - SessionStore / UserStore Protocol 정립 — Mongo/Redis impl 이 같은 surface 로 swap 가능
  - /admin/health admin-only gate 적용 — architecture.md §5.2 RBAC matrix 와 정합
  - Timing-safe + generic error message → user enumeration oracle 차단
  - Lazy expiry eviction → caller code 단순화
- 다음 세션 시작 포인트: TASK-002-4 (Mongo connection + users collection + admin seed script + indexes)
- 남은 리스크 및 후속 작업:
  - **auth/ LOC 누적 ~250 LOC** (4 모듈 × ~60): architecture.md §2.1 auth/ 예산 200 LOC 초과 ~25%. TASK-002-4/5 에서 추가 모듈 (MongoUserStore, RedisSessionStore) 가 합류 → ~350 LOC 예상, 약 75% 초과. 후속 cycle 에서 retrospective cleanup 검토
  - **/v1/auth/login timing attack 정밀 측정**: dummy verify 가 wrong pwd 분기 timing 과 비슷한지만, 실제 측정 없음. P1 에서 통계적 검증
  - **argon2 params**: PasswordHasher default (m=65536, t=3, p=4) 사용 — production 에서 tune 필요 여부 검토
  - **InMemoryUserStore.add_user** test helper: production 코드에 test helper — Mongo impl 추가 후 제거 검토
  - **admin_seed default password "changeme-please"**: docker-compose / .env.example 기본값 — production 부트스트랩 후 즉시 rotate 필요. CLI (TASK-002-7) 에서 강제 rotate 권장
  - **User enumeration via /v1/auth/login response time**: dummy verify 로 mitigation 했지만, *registration API* 도입 시 (P1) 추가 mitigation 필요
